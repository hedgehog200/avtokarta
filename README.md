# АВТОКарта

**Автоматизированная система учёта эксплуатации пожарных автомобилей МЧС**

Desktop-приложение (WPF / .NET Framework 4.5.2) для формирования эксплуатационных карточек пожарных автомобилей, расчёта норм расхода ГСМ и генерации отчётной документации в Excel.

---

## Модули

### Управление частями (подразделениями)
Справочник подразделений МЧС. Хранит наименование, номер, регион, ФИО начальника и старшего водителя, контактные данные. Поддержка нескольких частей с возможностью переключения между ними через боковую панель. Привязка тарифов расхода топлива к бакам (бензин / дизель) на уровне части.

### Управление автомобилями
Справочник пожарных автомобилей в рамках каждой части. Для каждого автомобиля: государственный знак, марка, тип (АЦ-3-40, АЦП 40 6/3 и др.), номер карточки, пробег шасси и двигателя, индивидуальные нормы расхода топлива и масел. Полный CRUD с валидацией и проверкой уникальности государственного знака.

### Эксплуатационная карточка
Основной рабочий экран. Ежедневные записи с автоматическим расчётом расхода топлива по формуле:

```
По_норме = (Пробег × Норма_пробега) + (С_насосом × Норма_насоса) + (Без_насоса × Норма_холостого_хода) + (Смена_караула × Норма_смены) + (Прочие × Норма_прочих)
```

Автоматическая классификация выездов (пожар, учения, ложный вызов, прочий пробег) по ключевым словам. Подсчёт итогов за месяц: факт, норма, экономия / перерасход. История карточек по месяцам. Учёт поступления топлива (централизованно, расчёт своего/чужого РЦ, закупка на местах).

### Склад ГСМ
Учёт поступления горюче-смазочных материалов: моторное масло, трансмиссионное масло, специальные жидкости, пластичная смазка. Для каждой партии: дата, марка, количество, номер документа, поставщик. Автоматический расчёт остатков по типам. Данные используются при генерации отчёта ГСМ.

### Отчёт ГСМ
Генерация комплекта отчётных документов в Excel (формат A4, Times New Roman) по шаблону МЧС:

| Лист | Содержание |
|------|------------|
| Титульный лист | Приложение №12 к приказу ГУ МЧС |
| Опись | Внутренняя опись документов дела |
| (1) Донесение | Отчёт движения ГСМ (бензин, дизтопливо, масла, жидкости) |
| (2) Расшифровка | Движение ГСМ при работе автотехники и мех. инструмента |
| (3) Временные показания | Расшифровка временных показаний и движения ГСМ |
| (4) Масла и жидкости | Движение масел, спец. жидкостей и пластичных смазок |
| Концевой лист | Подписи руководителя и старшего водителя |

Разделение данных по типам топлива (бензин / дизель), видам поступления, видам расхода. Автоматическое открытие файла после создания.

### Выгрузка в Excel (эксплуатационная карточка)
Экспорт содержимого карточки в формат `.xlsx` с полным форматированием:合并 ячеек, границы, жирный шрифт, авто-подбор ширины колонок, числа с 2 знаками после запятой.

### Передача данных
Экспорт / импорт всех данных (части, автомобили, карточки) в зашифрованный архив `.avto`. Архив содержит JSON-файлы, упакованные в ZIP и зашифрованные AES-256. Импорт с валидацией пароля, защита от path traversal, санитизация импортируемых данных. Резервное копирование одной кнопкой.

### Шифрование AES-256
Все локальные данные шифруются алгоритмом AES-256 (CBC + PKCS7). Ключ генерируется через PBKDF2 (50 000 итераций, случайный Salt). Пароль хранится в защищённом хранилище Windows (DPAPI). Поддержка смены пароля с полным перешифрованием всех данных.

### Настройки и безопасность
Смена пароля с валидацией (мин. 8 символов, заглавные + строчные + цифры). Автосохранение каждые 60 секунд + ручное сохранение (Ctrl+S). Блокировка при бездействии (5 минут).

### Автообновление
Проверка наличия обновлений на GitHub при запуске. Скачивание ZIP / EXE, валидация целостности (размер + сигнатура). Batch-скрипт обновления с откатом при ошибке. Защита от параллельных обновлений (Mutex). Логирование всех действий в `updater.log`. Ручная проверка обновлений в настройках.

## Релиз v1.1.0

### Что нового
- **Автообновление из GitHub** — при запуске приложение проверяет наличие новой версии и автоматически скачивает/устанавливает обновление
- **Ручная проверка обновлений** — в разделе «Настройки» добавлена кнопка «Проверить обновления»
- **Логирование обновлений** — все действия записываются в `%LocalAppData%\AVTOKarta\updater.log`

### Оптимизации
- **Кэширование ключей шифрования** — PBKDF2 выполняется 1 раз вместо 3 на операцию (300K итераций вместо 900K)
- **Асинхронные сохранения** — `SaveSquadsAsync`, `SaveVehiclesAsync`, `SaveCardAsync` не блокируют UI
- **Загрузка данных в фоне** — окно отображается сразу, данные загружаются асинхронно
- **Отложенная миграция** — шифрование HMAC выполняется после показа данных, не блокирует интерфейс
- **Маркер миграции** — повторная миграция пропускается при следующих запусках

### Исправления
- Исправлен зависание при запуске на слабых ПК
- Убрана зависимость от `Dispatcher` в `CacheService`
- Улучшена обработка ошибок в batch-скрипте обновления (откат при неудачной замене)
- Добавлена проверка целостности скачанного файла (размер + сигнатура ZIP)
- Защита от параллельных обновлений (Mutex)
